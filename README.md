# 仓库提交类型

* **feat**：提交新功能
* **fix**：修改了Bug
* **docs**：只修改了文档
* **style**：调整代码格式，未修改代码逻辑
* **refactor**：代码重构，没有修改Bug也没有添加新功能
* **perf**：性能优化，提高性能的代码更新
* **test**：添加或修改测试代码
